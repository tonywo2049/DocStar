# 合成需求（fixture）

## A. 功能需求

### R1 质押与验证者生命周期

- **R1-AC1** WHEN 账户在 Core 完成质押且满足准入 THEN 自下一 epoch 边界进入活跃验证者集合。
- **R1-AC2** WHEN 验证者发起退出 THEN 进入解绑期，期间其质押仍可被罚没。
- **R1-AC3** WHEN Core 停止出块 THEN 全部解绑计时暂停，恢复出块后自暂停点继续。

### R2 验证者集合跨链分发

- **R2-AC1** WHEN 集合换届 THEN 换届凭据分发至各链且可移植验证。

- **R3-AC1** WHEN 同一密钥跨链复用 THEN 每条签名绑定 `chain_id`，重放即拒收（本条与 C2-AC1 联动）。
